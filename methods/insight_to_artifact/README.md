# Insight-to-Artifact Method — Думка в перевірний об'єкт

> insight → definition → protocol → file → test → evidence

## definition
Перетворення інсайту в зовнішній, перевірний об'єкт (файл, код, схема, тест),
який існує незалежно від твоєї пам'яті та може бути перевірений іншим.

## input
Сирий інсайт — думка, що здається важливою, але ще не матеріалізована.

## method
1. **definition** — сформулювати інсайт одним точним реченням.
2. **protocol** — описати кроки застосування (input/method/output).
3. **file** — винести в артефакт за контрактом `schemas/artifact.schema.json`.
4. **test** — додати перевірку, що артефакт робить заявлене.
5. **evidence** — зафіксувати результат прогону як доказ.

## output
Прийнятий артефакт із 7 секціями + тест, що проходить.

## validation
- `tools/artifact_checker.py` → 0 відсутніх секцій.
- Є тест у `tests/`, що зелений.
- Без доказу (зелений тест / прогін) інсайт лишається гіпотезою, не фактом.

## example
Цей репозиторій — приклад: інсайт «межі мови = межі мислення» → 7 методів →
файли → тести (`make test`) → доказ (зелений прогін).

## failure_modes
- **Інсайт без артефакту**: лишається в голові → не перевіряється, забувається.
- **Артефакт без тесту**: заявляє, але не доводить → додай перевірку.
- **Доказ-театр**: тест перевіряє не те, що заявлено → звір тест із `definition`.
