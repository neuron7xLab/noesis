# Reflection Method — Дзеркало мислення

> Метод дзеркального опису власного мислення. LLM не додає нового — він повертає
> структуру твого запиту, щоб ти побачив намір ззовні.

## definition
Перетворення сирого, нечіткого наміру на кристалізований, готовий до дії, через
віддзеркалення його структури мовною моделлю.

## input
Сире повідомлення людини: думка, бажання, фрустрація, напівсформульований запит.

## method
1. Подати сирий намір у LLM з промптом `prompts/finalizer_mirror.md`.
2. LLM повертає п'ять шарів: явний запит, операція, ціль, обмеження, наступна дія.
3. Людина читає віддзеркалення й впізнає (або відкидає) свій намір.
4. Результат фіксується як артефакт за `schemas/reflection.schema.json`.

## output
JSON-об'єкт `reflection`: `raw_intent → crystallized_intent → blockers → next_action`.

## validation
- `tools/reflection_validator.py` → 0 помилок схеми.
- Операційний тест: після читання користувач **може одразу діяти** без додаткових питань.

## example
```json
{
  "raw_intent": "хочу нарешті щось зробити з усіма цими файлами про мислення",
  "crystallized_intent": "зібрати MVP-репозиторій методів мислення за єдиним контрактом",
  "blockers": ["файли розпорошені по DATA", "немає єдиного промта-дзеркала"],
  "next_action": "написати 7 методів за контрактом definition→failure_modes"
}
```

## failure_modes
- **Лестощі замість дзеркала**: модель підтверджує намір замість того, щоб показати його структуру → вимикай емпатію, лиши аналіз.
- **Галюцинований намір**: модель домислює ціль, якої немає → познач `[припущення: …]`.
- **Розмивання**: відповідь довша за вхід → застосуй Finalizer (90–110 слів).
