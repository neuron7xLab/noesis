# Extrapolated Thinking Method — Прогноз наслідків

> Прогнозування наслідків рішення в кількох часових горизонтах.

## definition
Розгортання одного рішення вздовж осі часу — зараз / день / тиждень / місяць /
рік — щоб побачити відкладені наслідки, які невидимі в момент вибору.

## input
Одне конкретне рішення або дія, яку розглядаєш.

## method
1. Зафіксувати рішення одним реченням.
2. Для кожного горизонту (`now → 1d → 1w → 1m → 1y`) описати очікуваний стан.
3. Позначити, де наслідки **розходяться** з наміром.
4. Якщо є горизонт із критичним ризиком — повернутися до Reverse Inference.

## output
Таблиця горизонтів `{ horizon: consequence }` + перелік точок розходження.

## validation
- Заповнені всі 5 горизонтів (порожній горизонт = сліпа пляма).
- Кожне розходження має або пом'якшення, або свідоме прийняття ризику.

## example
```text
Рішення: "запостити сирий чернетковий метод публічно сьогодні"
now: швидкий дофамін, є артефакт
1d:  перші коментарі, можливі правки
1w:  репутаційний слід від сирого формулювання
1m:  або забуто, або стало основою
1y:  визначає, чи серйозно сприймають увесь репозиторій
Розходження: 1w-1y ризик > 1d вигоди → спершу прогнати через Finalizer.
```

## failure_modes
- **Лінійна екстраполяція**: припускати, що тренд незмінний → познач невизначеність.
- **Ігнор хвоста**: оцінювати лише найближчий горизонт → 1m/1y обов'язкові.
- **Параліч**: занадто багато сценаріїв → 1 базовий + 1 ризиковий, не більше.
