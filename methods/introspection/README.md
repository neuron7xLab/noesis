# Introspection Method — Карта внутрішнього стану

> Метод внутрішнього розбору: причина, мотивація, блокер, бажаний результат.

## definition
Розділення емоційного або нечіткого запиту на чотири складові — намір, блокер,
обмеження, дія — щоб зникла неоднозначність наступного кроку.

## input
Емоційний, тривожний або заплутаний запит, де ціль змішана з шумом.

## method
1. Подати запит у LLM з промптом `prompts/introspection_engine.md`.
2. Розділити на: що я хочу / що мене блокує / чого я уникаю / що зробити першим.
3. Відокремити **сигнал** (намір) від **шуму** (емоція, контекст, виправдання).
4. Зафіксувати карту як артефакт рефлексії.

## output
Карта внутрішнього стану: `{ намір, блокер, обмеження, перша_дія }`.

## validation
- Неоднозначність наступного кроку **зникає**: є рівно одна перша дія.
- Блокер сформульований як усувна умова, а не як емоція.

## example
```text
Вхід:  "я знову нічого не встигаю і все валиться"
Намір:    закрити один конкретний результат сьогодні
Блокер:   немає пріоритету — все здається однаково терміновим
Обмеження: не починати нових гілок
Перша дія: вибрати 1 задачу й застосувати до неї Reverse Inference
```

## failure_modes
- **Псевдодіагноз**: модель ставить «діагнози» замість розбору → заборонено (`docs/ethics.md`).
- **Зациклення на емоції**: розбір тоне в почуттях → жорстко тягни до дії.
- **Множинні «перші дії»**: якщо їх більше одної — карта не зійшлася, повтори.
