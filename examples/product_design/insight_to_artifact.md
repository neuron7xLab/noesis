# Приклад: Insight-to-Artifact для дизайну продукту

**Інсайт:** «користувачі плутають намір з планом, тому формулюють задачі нечітко».

Перетворюємо в артефакт за контрактом:

```json
{
  "definition": "Onboarding-крок, що розділяє намір (навіщо) і план (як) перед першою задачею.",
  "input": "Сире формулювання задачі від користувача.",
  "method": "Промпт COGNITIVE_EXPANDER застосовується до ключового терміна задачі.",
  "output": "Картка: намір + план окремо.",
  "validation": "A/B: частка чітко сформульованих задач зростає проти контролю.",
  "example": "Задача 'зробити лендинг' → намір: 'залучити перших 10 клієнтів'.",
  "failure_modes": "Користувач пропускає крок; намір=план; занадто довгий onboarding."
}
```

Перевірка артефакту:
```python
from tools.artifact_checker import check_artifact, is_valid_artifact
assert is_valid_artifact(artifact)  # усі 7 секцій присутні
```
