# VERDICT — Cognitive Mirror Engine v0.2.0

Чесний аудит. Сепарація сигналу від шуму застосована до самого продукту.

## Definition of Done — виконано
> Система доводить, що вона не купа промптів, видаючи **валідований артефакт із
> сирого людського хаосу однією командою**.

```bash
python -m cme.cli pipeline examples/messy/03.txt --evidence out/
# → Evidence Bundle (6 файлів), validations_passed: true, одна наступна дія
```
10 messy real-world прикладів проходять трубу end-to-end; кожна валідація PASS
(`tests/test_cme_examples.py`). **60/60 pytest, mypy --strict (25 файлів), ruff.**

## Що працює
- Уніфікований `cme` CLI: mirror · introspect · reverse · artifact · pipeline · validate.
- FastAPI: /health /finalize /intent /mirror /introspect /reverse /artifact /pipeline /validate.
- Evidence Bundle: raw_input.md · mirror.md · artifact.json · validation.json · decision.md · manifest.json (sha256-id).
- Method Registry з name/use_case/trigger/input_schema/output_schema/failure_modes + селектор лінзи.
- Кожна труба завершується **однією конкретною наступною дією**.

## Що детерміноване (підлога, працює офлайн, завжди валідне)
- Вся **валідація**: word count, anchor terms, required fields, 7-секційна повнота,
  виконуваний фальсифікатор, forbidden claims, hallucination risk, soundness.
- Детермінований генератор mirror/introspection/artifact — евристичний розклад
  вхідного тексту в структуру; гарантує СТРУКТУРНО валідний артефакт без мережі.
- Reverse inference (goal regression), Method Registry, Evidence Bundle.

## Що потребує LLM (підсилення якості, не коректності)
- Природна якість фіналайзера й глибина дзеркала наміру (`--backend cli|sdk|llm`).
- LLM **пропонує**, детермінований гейт **розпоряджається**: якщо LLM-вихід не
  проходить контракт (швидкість + усі 6 якорів), система падає на детерміновану
  підлогу. Продукт ЗАВЖДИ видає валідований артефакт.

## Що валідоване (числом, не словом)
- Контракт фіналайзера: 90–110 слів + 6 якорів (`tools/finalizer100.py`).
- 7 секцій MethodArtifact непорожні + виконуваний фальсифікатор (Popper).
- Forbidden claims = 0; hallucination risk ≠ high.
- Goal-regression soundness: next_action = перша відсутня умова.
- Формальні інваріанти 7 методів (`formal/`, `docs/THEORY.md`).

## Що чесно фейкове / слабке (не приховую)
- **Детермінований mirror — це шаблон, не розуміння.** Він заповнює слоти
  евристикою (split речень, маркери емоції/блокера) і гарантує СТРУКТУРУ, не
  семантичну глибину. Глибина приходить лише з LLM-бекендом.
- **Anchor «намір» — строго літеральний.** LLM-фіналайзер часто семантично повний,
  але без слова «намір» → валідатор його завалює (бачено наживо). Це не баг гейта,
  а строгість контракту; детермінована підлога обходить це за конструкцією.
- **Method selection — груба евристика** (тригер-слова), не класифікатор.
- **Hallucination risk — груба проксі** (маркери надвпевненості + числа без джерела),
  не вимірювання фактичності.

## Що лишається дослідженням (не продакшн)
- Семантична валідність дзеркала (чи реконструйований намір = справжній намір користувача).
- Калібрування метакогнітивного γ на реальних (впевненість, коректність) парах.
- Чи справді вихід «менш емоційно шумний» — потрібна метрика афективного навантаження.
- Конструктна валідність форбіден/галюцинаційних гейтів на змагальних входах.

## Наступні 3 інженерні кроки
1. **LLM-бекенд для introspection та artifact** (зараз лише mirror) + контрактний
   фолбек на детерміновану підлогу, як у mirror.
2. **Адверсивний набір** з ~30 входів, що цілять у форбіден/галюцинацію/anchor-евейжн;
   гейти мусять ловити, інакше — посилити.
3. **Афективно-навантажувальна метрика** (лексикон + дельта) для кількісного доказу
   тези «менш емоційно шумний» — зараз це твердження без числа.
